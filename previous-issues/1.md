# Issue 1: Vim j/k navigation breaks after switching sidebar tabs or Cmd+Tab

## Symptom

After navigating between sidebar tabs (e.g. Cmd+3 then Cmd+2) or switching away from the app and back (Cmd+Tab), the `j`/`k` vim keys stopped navigating table rows. Other shortcuts like `?` (toggle hints) and `Shift+R` (refresh) continued to work.

## Root Cause

A React effect ordering issue between parent and child components.

**Parent** (`useVimNavigation` in `AppContent`):
```ts
useEffect(() => {
  useVimStore.getState().resetSelection(); // sets listLength: 0
}, [location.pathname]);
```

**Child** (`PRTable`):
```ts
useEffect(() => {
  useVimStore.getState().setListLength(visiblePRs.length); // sets listLength: N
}, [visiblePRs.length]);
```

React fires `useEffect` callbacks **children-first, then parents** within the same commit. On a route change:

1. Child `PRTable` mounts and its `useEffect` fires first: `setListLength(4)`
2. Parent `useVimNavigation`'s `useEffect` fires second: `resetSelection()` -> `listLength: 0`

The parent's reset wiped out the child's freshly-set `listLength`. With `listLength === 0`, `moveSelection()` bails out immediately (`if (listLength === 0) return`), so `j`/`k` did nothing.

Shortcuts like `?` still worked because `toggleHints()` is a direct store action that doesn't depend on `listLength`. `Shift+R` (refresh) sometimes fixed it because the data re-fetch triggered a re-render of `PRTable`, which called `setListLength` again after the parent's reset had already completed.

## Fix

Changed `useEffect` to `useLayoutEffect` for the `resetSelection()` call in `useVimNavigation.ts`.

`useLayoutEffect` fires synchronously after DOM mutations, before any `useEffect` callbacks. This guarantees the ordering:

1. Parent `useLayoutEffect`: `resetSelection()` -> `listLength: 0`
2. Child `useEffect`: `setListLength(4)` -> `listLength: 4`

## Files Changed

- `frontend/src/hooks/useVimNavigation.ts` - Changed `useEffect` to `useLayoutEffect` for the `resetSelection` call on `[location.pathname]`.

## Related Context

- `frontend/src/stores/vimStore.ts` - `resetSelection()` zeros both `selectedIndex` and `listLength`.
- `frontend/src/components/pr/PRTable.tsx` - `setListLength(visiblePRs.length)` syncs the vim store with the table's visible row count.
- `frontend/src/hooks/useWindowFocus.ts` - Separate fix for WKWebView losing DOM focus after Cmd+Tab in production Wails builds (refocuses `document.body`).
